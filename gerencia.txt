1. Workflow
O workflow em desenvolvimento de software é o fluxo de trabalho que organiza como as tarefas são realizadas, desde a implementação até a entrega final. 
Ele define etapas claras para que desenvolvedores possam trabalhar de forma colaborativa, garantindo que funcionalidades sejam desenvolvidas, testadas, 
revisadas e integradas corretamente. Exemplos comuns incluem o centralizado, onde todos os desenvolvedores trabalham em uma única branch, e o feature 
branch, onde cada tarefa tem sua própria branch isolada para evitar conflitos.

2. Issue Tracking
O Issue Tracking é um sistema utilizado para gerenciar problemas, tarefas ou mudanças em um projeto de software. Cada issue representa uma unidade de 
trabalho, como uma nova funcionalidade, correção de bug ou melhoria. Cada issue é geralmente categorizada com atributos como prioridade, descrição e 
status, facilitando o acompanhamento do progresso e permitindo a colaboração eficiente entre equipes.

3. Controle de Mudança
O Controle de Mudança é o processo de gerenciar e autorizar alterações em um software de forma organizada. Ele envolve a proposição, revisão e 
aprovação de mudanças para garantir que qualquer modificação no código não afete a estabilidade do sistema. Isso é feito para evitar que erros sejam 
introduzidos, garantindo que as mudanças sejam planejadas e coordenadas.

4. Controle de Versão
O Controle de Versão permite gerenciar e documentar todas as mudanças feitas no código-fonte ao longo do tempo. Ele permite que várias versões do 
software coexistam, com histórico completo de modificações e a capacidade de reverter para versões anteriores, se necessário. O sistema de controle 
de versão mais utilizado atualmente é o Git, que facilita o trabalho em equipe e o desenvolvimento distribuído.

5. Milestones
Milestones são marcos de projeto que agrupam um conjunto de issues com um objetivo específico. Eles ajudam a organizar o desenvolvimento em fases, 
permitindo que a equipe tenha metas claras, como a finalização de um conjunto de funcionalidades. Cada milestone é um ponto de verificação para medir
o progresso em direção a uma meta maior.

6. Baseline
A Baseline é uma versão estável e aprovada de um item de configuração, como o código de uma aplicação ou um artefato de software. Ela representa um 
ponto no desenvolvimento onde todas as mudanças foram revisadas e aceitas, servindo como uma referência para futuras implementações ou correções.

7. Auditoria de Configuração
A Auditoria de Configuração é o processo de revisar as alterações feitas em um software para garantir que estejam corretamente documentadas e que o 
sistema está funcionando de acordo com as especificações definidas. Ela garante que a versão final seja rastreável e que todos os requisitos e mudanças 
foram atendidos de forma correta e controlada.

8. Merge Request / Pull Request
O Merge Request ou Pull Request é um mecanismo utilizado em plataformas como GitHub e GitLab para solicitar a integração de mudanças feitas em uma 
branch com outra, normalmente a branch principal. Antes de realizar o merge, as mudanças são revisadas por outros membros da equipe, garantindo que o 
código esteja correto, testado e pronto para ser integrado ao projeto.
